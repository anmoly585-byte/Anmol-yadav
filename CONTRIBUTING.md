# Contributing Guidelines

## How to Contribute

We welcome contributions from developers of all skill levels!

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/Anmol-yadav.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make changes and test them
5. Commit: `git commit -m "Add feature description"`
6. Push: `git push origin feature/your-feature`
7. Create a Pull Request

## Code Style Guidelines

### Java
```java
// Use camelCase for variables and methods
private String userName;

public void processTransaction() {
    // Implementation
}

// Add comments for complex logic
// One line comment for simple explanations

/**
 * Multi-line comments for methods
 * @param parameter Description
 * @return Description
 */
```

### JavaScript
```javascript
// Use const/let instead of var
const MAX_AMOUNT = 100000;

// Use meaningful variable names
const userBalance = 50000;

// Use arrow functions where appropriate
const calculateFee = (amount) => amount * 0.01;
```

### HTML/CSS
```html
<!-- Use semantic HTML -->
<main>
    <section class="account-section">
        <h2>Account Details</h2>
    </section>
</main>

/* Use meaningful class names */
.account-card { }
.transfer-form { }
```

## Testing

Before submitting a PR:

1. Test in Chrome, Firefox, Safari
2. Check mobile responsiveness
3. Verify all links work
4. Check console for errors
5. Test Java compilation (if modified)

## Commit Messages

Use clear, descriptive commit messages:

```
✨ Add new feature description
🐛 Fix bug description
📝 Update documentation
🎨 Improve code formatting
♻️ Refactor code
🔒 Add security feature
⚡ Improve performance
```

## Pull Request Process

1. Update README if adding new features
2. Add comments to complex code
3. Ensure no console errors
4. Test thoroughly
5. Write descriptive PR title and description
6. Link related issues

## Reporting Bugs

Use the GitHub Issues tab:

1. Check if bug already reported
2. Include browser and OS information
3. Describe steps to reproduce
4. Include screenshot/error message
5. Suggest fix if possible

## Feature Requests

1. Use GitHub Issues
2. Describe use case
3. Explain expected behavior
4. Provide examples if possible
